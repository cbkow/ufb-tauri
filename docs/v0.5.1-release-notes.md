# v0.5.1 release notes + carryover list

Released 2026-04-18.

## Major changes

### WinFsp port (Windows)
- ProjFS removed entirely; Windows sync backend is now WinFsp 2.1.25156 via the SnowflakePowered `winfsp` 0.12 crate.
- Installer bundles the WinFsp runtime MSI; silent-installs on first run if not already present.
- UFB relicensed AGPL-3.0 → GPL-3.0 to satisfy WinFsp's FLOSS exception.
- All 6 slices from `docs/windows-winfsp-port-plan.md` complete:
  - Slice 1: server production-ized + wired to main.rs
  - Slice 2: metadata cache authority
  - Slice 3: block-level content cache
  - Slice 4: write path + conflict detection
  - Slice 5: installer + service integration
  - Slice 6: compat burn-in (rolled in with user testing)

### Sync lifecycle + cache-dir
- `MacosCache` / `CacheIndex` now take a `cache_dir: &Path` parameter — the `syncCacheRoot` setting is finally honored on both OSes (it was persisted but ignored prior to this release).
- Clean shutdown plumbing: both NFS and WinFsp server `start()` functions return a `SyncServerHandle` with `shutdown_and_wait()`. No more "server lives forever, drops on process exit."
- Live cache-root switch: changing the cache directory in Settings tears down all sync servers, drops the `SharedCaches` Arc, and rebuilds at the new path — no agent restart needed.
- Tray Stop/Start/Restart on a sync mount now actually stops/starts/restarts the WinFsp (or NFS) server, not just the SMB session half.
- `MountService` route_event intercepts sync-mount events and coordinates server lifecycle via the respawn channel (`Option<String>` payload: `None` = all, `Some(domain)` = single).

### Windows build / install
- `[profile.dev.package."*"] opt-level = 2` in `src-tauri/Cargo.toml` — fixes "tauri dev is 50× slower than release" for CPU-bound deps (`image` crate JPEG decode in particular). Root cause of user-reported "thumbnails load 2 at a time, long pauses."
- `.iss` updated to pull agent from `target/x86_64-pc-windows-msvc/release/` (not `target/release/`) — previous mismatch was shipping stale agents.
- `winfsp` crate `"debug"` feature removed. The feature calls `GetStdHandle(STD_ERROR_HANDLE).unwrap()` at `FileSystemHost::new()` time — panics when launched without a console (Run-key autostart = every installed-app launch). Blocking bug for sync in release before this.
- Probe-first SMB session establishment in `orchestrator.rs::start_sync` — try `fs::metadata` first, only fall back to `WNetAddConnection2W` if unreachable. Fixes spurious `ERROR_LOGON_FAILURE` (1326) when Windows has a cached session with different credentials.
- Explorer sidebar nav-pin registration now includes sync mounts (pre-WinFsp, sync mounts had their own CF-API shell extension registration and were explicitly skipped in `explorer_pins.rs`).
- Vendored WinFsp MSI at `installer/vendor/winfsp-2.1.25156.msi` (gitignored via `installer/vendor/*.msi`).

### UI
- New **tree view** (Finder/VS-Code-style disclosure triangles) as a third view mode alongside list + grid. Session-scoped expansion + cache. Per-level sort matches list-view settings. Spinner for slow loads with 150ms delay so fast folders don't flicker.
- **Resizable columns** on every column in both list-view browsers (Name + Size + Modified + Type + Synced) and item-list panels (Name + metadata + Modified). Per-instance persistence via localStorage.
- **256px Windows system icons** for grid view (SHIL_JUMBO via `IImageList`) with bucketed cache keys so grid requests don't starve behind list-view's 32px cache entries.
- **Nilesoft Shell integration**: removed "Union Apps" taskbar folder (app locations in flux, no longer useful). Restored `LucidLink` + `IO` entries to `Union Goto` menu that had been dropped during an earlier path cleanup.
- Grid-size slider no longer clips in narrow browser panes — `.search-input` now has `min-width: 60px` so it yields space to the slider, and `.sort-control` has `flex-shrink: 0`.

### Bugs fixed
- **Windows drag-out OLE leak** (`src-tauri/src/drag_out.rs`): `HDropDataObject` now has a `Drop` impl that `GlobalFree`s the HGLOBAL. Symptom was system-wide drag-drop freezing after one drag; see `docs/windows-drag-out-notes.md`.
- **Thumbnail traffic jam** (`src/stores/fileStore.ts`): `clearThumbnailQueue` was resolving pending promises with `null` on every 2-second agent heartbeat, polluting `noThumbPaths`. Fix: only clear on fresh navigation, not on refresh. Plus `reconcile` on entries replace so `<For>` preserves `ThumbnailImage` instances across heartbeats.
- **Tree view scroll reset** (`src/stores/fileStore.ts`): row-identity cache in `treeList()` so stable rows keep the same reference across renders. Solid's `<For>` reconciles by ref; without the cache, every tree refresh unmounted+remounted every row, scrolling to top.
- **Tree view selection pruning**: refresh's valid-paths check now includes `childrenCache` so deep selections survive heartbeats.
- **Shift-range selection**: now walks the visible ordered list (tree-list in tree mode, sortedEntries in list/grid), not the raw backend-sort `entries`.
- **NFS delete in macOS sync volumes**: `remove_dir` against a folder containing filtered hidden junk (`.DS_Store`, `@eaDir`, `#recycle`) was failing with ENOTEMPTY. Now sweeps hidden children before retrying `remove_dir`. User-visible entries still return `NFS3ERR_NOTEMPTY` honestly — no `remove_dir_all` foot-gun.

## Carryover / known issues

### Windows `ERROR_UNTRUSTED_MOUNT_POINT` on first launch
See `docs/windows-smartscreen-448.md`. SmartScreen scans unsigned EXEs in a restricted integrity context on first run; reparse-point traversal is blocked during the scan. Retry helper in `file_ops.rs` doesn't fully resolve it because the scan takes longer than our retry window.

**Real fix**: code-sign the binaries. Not pursued — cert cost.

**User workaround**: quit and relaunch UFB after first install. Second launch works reliably.

### UAC elevation for symlink creation
Non-sync mounts create symlinks at `C:\Volumes\ufb\{share}` pointing at UNC targets. Requires either admin or Developer Mode. If neither is set, agent exposes `needsElevation` state; frontend shows a "Mount Volumes" button that triggers `ShellExecuteW runas`. Works but is an extra manual step on machines without Dev Mode.

### macOS QuickLook thumbnail fallback
Not implemented. If the `image` crate fails on an unusual image format on macOS, there's no fallback. TODO comment in `thumbnails.rs::try_extract`. Implement via `QLThumbnailGenerator` (requires `objc2_quick_look_thumbnailing` or manual FFI) when we hit a real case.

### Pixel JPEG decoder edge cases
`image` crate rejects some Google Pixel Motion Photo / HDR / MACRO_FOCUS JPEGs. Tried: Windows Shell fallback (returns stylized previews for non-image types — rejected), ffmpeg fallback (didn't help). Reverted to just the `image` crate. Haven't found a reliable fix that doesn't drag in additional complexity.

### Tauri CSP warnings in release
`Fetch API cannot load http://ipc.localhost/plugin%3Aevent%7Clisten` — Tauri's primary IPC route blocked by CSP, falls back to `postMessage`. Functional but slower initial loads. Fix: add `http://ipc.localhost` to CSP `connect-src` in `tauri.conf.json`. Low priority.

### Reachability probe (disabled)
`fs::metadata` on reparse points is an unreliable liveness signal — false negatives during startup, stale state, etc. Frontend probe + "Unavailable" visual state disabled. Plumbing (command, store methods, CSS class) kept so we can re-enable with a smarter implementation later (probably agent-side, sharing the NasHealth probe signal).

### Mesh sync snapshot path error
Log shows `Failed to register endpoint: Failed to create node dir: failed to create whole tree` every few seconds when the configured farm path `\\192.168.40.100\MinRender\ufb` isn't reachable. Unrelated to this release but noisy in the log. Defer to when the farm is more stable.

## Test matrix

Verified working in v0.5.1:
- Windows install from `ufb-tauri-setup-0.5.1.exe` → agent autostarts via Run-key → Tauri launches → all 3 mounts up
- WinFsp mount served end-to-end at `C:\Volumes\ufb\Jobs_Live` (media file scrub, block-level caching, freshness sweep)
- Non-sync UNC symlinks at `C:\Volumes\ufb\{FLAME_JOBS,GFX_Dropbox}` (when NASes reachable)
- Tree view expand/collapse with 500+ item folders
- Drag-out from UFB to Explorer (single file + multi-file)
- 256px icons in grid view
- Tray Stop/Start/Restart on each mount type
- Cache-dir change via Settings → drains + respawns cleanly

Known limitations:
- First launch after install: see SmartScreen section above
- Unsigned binary: Windows Defender / SmartScreen may warn on install
- Agent UAC prompt on machines without Developer Mode
